# QA / اختبار سريع

## تم التحقق منه داخل البيئة

- تشغيل سيرفر محلي `python -m http.server` وفتح الملفات بنجاح.
- التأكد من وصول صفحات HTML (200 OK).
- التأكد من وصول ملفات JSON في `assets/content/` (200 OK).
- فحص Syntax لملفات JS الأساسية عبر `node --check` (بدون أخطاء).

## Checklist اختبار يدوي (قبل النشر)

1) الرئيسية
- عرض التعريف + آية/حديث (مع المرجع)
- زر التثبيت (Install) يعمل على الأجهزة الداعمة
- زر مشاركة البرنامج
- عرض التقييمات/قصص النجاح

2) بدء الاختبار
- حفظ بيانات المستخدم
- الانتقال لـ `test.html`

3) الاختبار الكامل
- 50 سؤال + تنقل + شبكة أسئلة
- تشغيل/إيقاف الشرح
- حفظ التقدم عند التحديث
- إنهاء الاختبار → تحويل للنتائج
- بعد الإنهاء: قفل 24 ساعة + عدّاد واضح

4) النتائج والخطة
- عرض التحليل + نقاط الضعف
- توليد خطة + جدول (من index_course.json)
- زر نسخ الخطة + مشاركة الخطة
- زر تحميل PDF (طباعة/Save as PDF)
- زر/رابط موقع الدورة المكثفة
- CTA لمجموعة الالتزام

5) الكويزات
- توليد كويز حسب (القسم/العدد/الصعوبة/النموذج/الأكثر تكرارًا)
- Seed في الرابط يعيد نفس الكويز
- تشغيل/إيقاف الشرح
- نتيجة + مراجعة الأخطاء
- مشاركة الكويز

6) الإشعارات
- Toast كل ~60–90 ثانية
- تتوقف داخل الاختبار والكويز أثناء الحل

7) السجل
- حفظ المحاولات + الكويزات
- مسح بيانات المستخدم

8) PWA
- manifest + service worker
- Offline fallback
